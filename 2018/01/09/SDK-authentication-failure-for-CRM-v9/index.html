SDK authentication failure for CRM v9

In CRM v9.0, when trying to pass the authentication via CRM SDK, we find we are unable to retrieve the OrganizationService. As the screenshot, the service is null, the authentication hasnâ€™t been passed.

By checking the fiddler trace, we can find the TLS 1.0 was used for the connection.

Reason

Why this happens because .NET Framework 4.5.2 or less makes HTTP connections with TLS 1.0 while CRM 9.0 organizations are blocking those calls.
Refer to https://blogs.msdn.microsoft.com/crm/2017/09/28/updates-coming-to-dynamics-365-customer-engagement-connection-security/

Resolution

  • Option 1:

    Upgrade the .NET Framework to 4.6.2 or higher

  • Option 2:

    Add below code before making the OAuth calls as I commented in the first image.

    ServicePointManager.SecurityProtocol = (SecurityProtocolType)3072;
